◆ mutex_lock()

rcode mutex_lock ( struct Mutex *  m)

Locks the mutex, blocking the calling thread if necessary.

If the mutex is already locked by another thread, the calling thread will wait until it becomes available. Recursive locking by the same thread is not allowed.

Parameters
mPointer to the mutex to lock.
Returns
  • 0: Lock acquired successfully.
  • DESCENT_ERROR_NULL: m is NULL.
  • DESCENT_ERROR_FORBIDDEN: Calling thread is invalid.
  • THREAD_ERROR_DEADLOCK: Calling thread already owns the mutex.

◆ mutex_timedlock()

rcode mutex_timedlock ( struct Mutex *  m,
uint64_t  nanoseconds 
)

Attempts to lock the mutex, blocking up to the specified timeout.

If the mutex is already locked by another thread, the calling thread will wait until it becomes available. Recursive locking by the same thread is not allowed.

Timeouts are capped at the value returned by time_max_timeout().

Parameters
mPointer to the mutex to lock.
nanosecondsThe maximum number of nanoseconds to wait.
Returns
  • 0: Lock acquired successfully.
  • THREAD_INFO_TIMEOUT: Lock could not be acquired within the timeout.
  • DESCENT_ERROR_NULL: m is NULL.
  • DESCENT_ERROR_FORBIDDEN: Calling thread is invalid.
  • THREAD_ERROR_DEADLOCK: Calling thread already owns the mutex.

◆ mutex_trylock()

rcode mutex_trylock ( struct Mutex *  m)

Attempts to lock the mutex without blocking.

If the mutex is already locked by another thread, this function returns immediately. Recursive locking by the same thread is not allowed.

Parameters
mPointer to the mutex to try locking.
Returns
  • 0: Lock acquired successfully.
  • THREAD_INFO_BUSY: Mutex is already locked by another thread.
  • DESCENT_ERROR_NULL: m is NULL.
  • DESCENT_ERROR_FORBIDDEN: Calling thread is invalid.

◆ mutex_unlock()

rcode mutex_unlock ( struct Mutex *  m)

Unlocks a previously locked mutex.

Only the thread that owns the mutex may unlock it. Unlocking a mutex not held by the calling thread will fail.

Parameters
mPointer to the mutex to unlock.
Returns
  • 0: Mutex unlocked successfully.
  • DESCENT_ERROR_NULL: m is NULL.
  • DESCENT_ERROR_FORBIDDEN: Mutex is not owned by the calling thread.

◆ mutex_wait()

rcode mutex_wait ( struct Mutex *  m,
struct Condition *  c 
)

Waits indefinitely on a condition variable while holding the mutex.

The calling thread must hold the mutex before calling this function. The mutex is released while waiting and re-acquired before returning.

Parameters
mPointer to the mutex currently held.
cPointer to the condition variable to wait on.
Returns
  • 0: Successfully waited and re-acquired the mutex.
  • DESCENT_ERROR_NULL: Either m or c is NULL.
  • DESCENT_ERROR_FORBIDDEN: Calling thread is invalid.
  • Other mutex errors: Any error returned by mutex_unlock or mutex_lock.
Note
Spurious wakeups are possible; always check the associated predicate.
