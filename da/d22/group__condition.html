Detailed Description

Thread synchronization via condition variables.

Condition variables allow threads to wait for arbitrary conditions to become true while releasing an associated lock. A waiting thread will atomically release the lock and suspend execution until it is signaled, then re-acquire the lock before returning.

Condition variables do not carry state. A signal or broadcast has no effect if no threads are waiting at the time it is issued.

All waits may experience spurious wakeups. Callers must always re-check the associated condition predicate after waking.

Signaling a condition variable establishes a happens-before relationship with threads that successfully return from a corresponding wait.

This mechanism is intra-process only and cannot be shared between processes.
